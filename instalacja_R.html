Instalacja środowiska R – wersja rozszerzona.

Jakub J. Guzek (jakub.j.guzek@gmail.com)

Wstęp

Poradnik ten przeznaczony jest dla osób, które wcześniej nie miały styczności z językiem \(\mathsf{R}\). Nie jest on przeznaczony dla osób, które mają doświadczenie z tym językiem, ani dla osób, które mają doświadczenie z analogicznymi środowiskami. Na wstępnie również musi zostać zaznaczone, że autor tego poradnika nie jest specjalistą, ani nawet doświadczonym użytkownikiem \(\mathsf{R}\), a tekst ten dotyczy tylko instalacji środowiska, a nie jest wprowadzeniem do niego.

Dobrym źródłem informacji na temat \(\mathsf{R}\) jest r-project, Stack Overflow, a dla użytkowników systemów UNIX-based (Linux, OSX) tak zwane manual pages dotyczące \(\mathsf{R}\), do których dostęp można uzyskać wpisując

man R

w terminalu1, a także ?[nazwa funkcji] w samym \(\mathsf{R}\)

Instalacja dla systemu Windows

Pobranie i instalacja

Ten krok jest dosyć prosty – należy wejść na stronę R Project (do której link zamieszczony został we wstępie do tego poradnika), następnie wybrać CRAN z menu, które powinno znajdować się po lewej stronie. Użytkownik zostanie wówczas przekierowany do wyboru serwera. Dla użytkowników pobierających z polski polecany jest jeden z serwerów niemieckich2 lub 0-Cloud. Kolejnym krokiem jest pobranie instalatora dla Windowsa (wybrać opcję \(\mathtt{base}\)). Następnie należy przeprowadzić standardową instalację.

Konfiguracja

Po uruchomieniu \(\mathsf{R}\) należy w okienku wpisać następujące komendy, aby zainstalować R Commander.

install.packages{"Rcmdr", dependancies = TRUE}
install.packages{"nFactors", dependancies = TRUE}
install.packages{"clusterSim", dependancies = TRUE}
isntall.packages{"mvShapiroTest", dependacnies = TRUE}
install.packages{"agricolae", dependancies = TRUE}

Instalacja dla systemu OSX (macOS)

Instalacja dla systemu OSX jest analogiczna jak dla sytemu Windows, należy przy pobieraniu rzecz jasna wybrać opcję dla macOS i pobrać binary dla użytkowników (a nie znajdującą się poniżej opcję dla developera). Następnie przeprowadzić standardową instalację

Alternatywą jest użycie package managera dla systemu macOS – homebrew Wówczas, należy wpisać 

brew install r

w terminal.

Konfiguracja

Najszybszą opcją jest uruchomienie \(\mathsf{R}\) w terminalu wpisując R i wpisanie poniższych komend.

install.packages{"Rcmdr", dependancies = TRUE}
install.packages{"nFactors", dependancies = TRUE}
install.packages{"clusterSim", dependancies = TRUE}
isntall.packages{"mvShapiroTest", dependacnies = TRUE}
install.packages{"agricolae", dependancies = TRUE}

Powyższe komendy mogą zwracać błąd dotyczący paczki tcl-tk, wówczas należy użyć homebrew i wpisać w terminal brew install tcl-tk

Instalacja dla systemów Linux

Instalacja dla dystrybucji Debian-based

sudo apt-get r-base

Instalacja dla dystrybucji Fedora

yum install -y R

Instalacja dla RHEL

sudo yum groupinstall -y ‘Development Tools’
sudo yum install -y libcurl-devel openssl-devel libxml2-devel
sudo yum install -y R

Instalacja dla dystrybucji Arch-based

sudo pacman -S r

Konfiguracja

Należy uruchomić terminal i wpisać \(\mathsf{R}\), a w samym \(\mathsf{R}\):

install.packages{"Rcmdr", dependancies = TRUE}
install.packages{"nFactors", dependancies = TRUE}
install.packages{"clusterSim", dependancies = TRUE}
isntall.packages{"mvShapiroTest", dependacnies = TRUE}
install.packages{"agricolae", dependancies = TRUE}

Troubleshooting

Jednym z najczęstszych problemów przy instalowaniu paczek w \(\mathsf{R}\) na systemach UNIX-like (OSX, Linux) jest błąd dotyczący braku tcl-tk lub błąd dotyczący braku kompilatora dla języka FORTRAN.

Brak tcl-tk

Na OSX problem ten najłatwiej rozwiązać instalując formułę tcl-tk przez homebrew, co można uczynić wpisując

brew install tcl-tk

w terminal.

Analogicznie na systemach linux należy pobrać paczkę tcl-tk przy użyciu package managera danego systemu.

Brak kompilatora dla języka FORTRAN

Na OSX problem ten również najłatwiej jest rozwiązać przy użyciu homebrew Jeżeli kompilator nie jest zainstalowany

brew install gfortran

Jeżeli kompilator jest zainstalowany, a mimo to występuje ten błąd

brew upgrade gfortran

Na systemach linux problem ten można rozwiązać instalując front-end’owy kompilator dla fortran’u od GNU – gcc-fortran

Przykład dla systemów Arch-based

sudo pacman -S gcc-fortran
